# CLAUDE.md（国贸期货杯·夺奖项目）

> 本文件继承全局 `~/.claude/CLAUDE.md` 全部规则，并补充本项目专属约束。冲突时以本文件为准。

---

## 项目身份

**第二届国贸期货杯·厦门大学**——模拟期货交易比赛
- 100 万虚拟金，禁出入金
- 交易品种：上期所/大商所/郑商所/中金所/能源中心/广期所 全部期货+期权品种
- 手续费 ×2、保证金 +3%
- 最少 10 个交易日才有效成绩；穿仓即出局
- 评分公式：收益率 40% + 最大回撤 20% + 夏普 30% + 波动率 10%

详细规则见 `rules/比赛规则.md`，公式数学拆解见 `rules/评分公式拆解.md`。

---

## 用户档案（重要）

- **完全零基础**：未做过实盘期货，需从合约/保证金讲起。任何术语首次出现必须用白话解释。
- **目标**：拿一等奖（前 4）。
- **时间**：可全天投入，已愿意付 LLM API Key 跑盯盘工具。

## 项目级硬约束（不可被任何指令覆盖）

1. **每一笔实际下单由用户本人在比赛官方软件上完成**——Claude 只给建议，绝不替用户操作账户、绝不索要账号密码、绝不接管账户。
2. **绝对禁止给出会导致穿仓的建议**：单品种保证金占用上限 30%、总保证金占用上限 70%、无止损单不下、马丁加仓严禁。
3. **策略方向 = 受控激进**（基础仓 30-40%，高确信信号叠加 60-70%），不是无脑梭哈。**评分公式偏爱低波动**，这是数学事实不是保守建议。
4. **诚实底线**：
   - 对某个信号没把握必须说"不确定"，禁止编造确定性判断
   - 报错必须拿编译/数据/截图证据，不许猜
   - 输了的单复盘必须诚实归因到决策错误，不许甩锅"市场不理性"
5. **不抓盗版书 / 不绕版权**：知识库只用维基百科、Goodreads、SSRN、公版书、公开博客、训练记忆，违者立刻停。
6. **API Key 绝不硬编码**：用 `.env`（在 `tools/.gitignore` 中），任何 commit 前 grep 一次 sk- 防泄露。
7. **预测/收益率绝不承诺**：本计划只能"提高前 4 概率"，不能"保证前 4"。
8. **先读书再回答**：每次用户提问交易/策略/品种相关问题，必须先读 `books/_consensus.md`（20条铁律）和相关品种手册，用知识库内容作为回答依据。禁止脱离知识库瞎猜。云端扫描器已嵌入 `tools/research/knowledge_base.py`（221条规则覆盖96本书），所有自动决策必须基于此知识库。
9. **主动探寻机会**：不能只等用户指令。每天主动扫描市场，用100本书的知识寻找潜在机会，有发现立即推送。像巴菲特一样研究，不是等别人告诉你买什么。

## 每日工作流

- **08:00** 跑 `tools/briefing/morning.py` → 输出 `daily/YYYY-MM-DD.md` 盘前简报
- **08:45** 用户阅读简报 → 与 Claude 对齐当日计划 → 用户拍板
- **09:00-15:00** Claude 后台跑 `tools/monitor/live_watch.py`，触发关键信号桌面通知
- **15:00** 收盘 → 跑 `tools/briefing/evening.py` → 复盘 + 明日预案
- **21:00** 夜盘前再来一轮简报（金属/能化/股指夜盘品种）
- **每周日** 周复盘 + 评分公式自估得分追踪
