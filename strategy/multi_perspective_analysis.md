# 多视角分析框架

> 每次重大交易决策前，必须启动至少 3 个视角 Agent 并行分析，综合后再做决定。

---

## 为什么需要多视角？

单一视角容易盲点：
- 只看技术面 → 忽略宏观政策转向
- 只看基本面 → 忽略资金博弈和短期情绪
- 只看新闻 → 忽略价格已反映预期
- 只看风控 → 可能过度保守错失机会

**100 本书共识（_consensus.md）**：
- #9 "不要预测，要准备" → 多视角就是准备
- #14 "亏钱是因为看到了想看的" → 多视角对抗确认偏差
- #15 "市场是非线性的" → 不同视角捕捉不同维度

## 5 个标准视角

### 1. 经济学家（宏观面）
- 关注：GDP、PMI、财政/货币政策、利率、汇率、产业政策
- 问题：经济周期在什么阶段？政策方向对需求意味着什么？
- 数据源：统计局、央行、财政部公告

### 2. 技术分析师（图表面）
- 关注：K线形态、均线、MACD、RSI、布林、成交量、持仓量
- 问题：趋势方向？关键支撑阻力？量价配合？
- 数据源：akshare K线数据、indicators/compute.py

### 3. 基本面分析师（供需面）
- 关注：库存、产量、开工率、进出口、季节性、产业链利润
- 问题：供需平衡表偏紧还是偏松？库存周期在哪阶段？
- 数据源：mysteel.com、统计局产量数据

### 4. 风险经理（风控面）
- 关注：仓位大小、止损位置、相关性、最大回撤、评分公式影响
- 问题：这笔交易对组合得分的影响？最坏情况是什么？
- 数据源：position_calc.py、score_simulator.py

### 5. 交易心理学家（情绪面）
- 关注：市场情绪、机构持仓变化、散户行为、新闻舆论
- 问题：市场情绪极端吗？机构在做什么？是否有反向指标？
- 数据源：mysteel 持仓日报、新闻情绪

## 执行流程

```
1. 收集数据（WebFetch 抓新闻 + akshare 拉行情 + indicators 计算信号）
2. 并行启动 3-5 个 Agent（每个视角一个）
3. 各视角独立给出判断：支持/中性/不支持 + 理由
4. Coordinator 综合裁决：
   - ≥3 个视角支持 → 可以执行
   - 2 支持 + 1-2 中性 → 谨慎执行（减半仓位）
   - ≤1 支持 或 有反对 → 不执行
5. 记录到 daily/ 日志 + lessons_learned.md
```

---

## 输出格式

每个视角 Agent 必须输出：
```
【视角名称】判断：支持/中性/不支持
核心理由（1-2句）
关键风险点
```
